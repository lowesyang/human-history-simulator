> _如果你拥有改写历史的力量，你会将人类带向何方？_

**Human History Simulator** 是一个以 LLM 多 Agent 系统为历史推演引擎的文明模拟器。从 **19 个纪元**中任选其一，纵跨 **3,600 年**，从前 1600 年的青铜熔炉到 2000 年的数字网络，在交互式世界地图上驱动 **1,400+ 文明**同时运转。每个文明都是深度建模的实体，追踪 **100+ 项状态字段**：GDP、军事力量、识字率、贸易路线、文化产出等等。每一回合，多 Agent 推演引擎编织新事件、计算状态转移、重塑地缘政治格局。没有两次推演会走向相同的历史。

疆域边界来自 **47 组真实历史 GeoJSON 快照**（前 2000 年 – 2010 年），基于开源学术地图构建，是真正的学术疆域数据而非近似轮廓。战争重划边界。贸易路线跨大陆转移财富。瘟疫吞噬人口。发明点燃变革。每一次变化都以字段级粒度记录，赋予你跨越千年的上帝视角来观察因果链。相同起点，无限分叉的历史。

## 特性

- **[19 个历史纪元](#支持的纪元)** 从青铜时代到现代世界，每个纪元预置历史准确的文明、统治者与地缘格局。
- **1,400+ 文明** 包括帝国、王国、城邦、部落、贸易网络，每个纪元同时模拟 60–100 个区域。
- **多 Agent 驱动演化** AI 编排器聚类区域、生成事件，并跨经济、军事、外交、文化等维度计算逐字段的状态转移。
- **真实历史疆域** 47 组 GeoJSON 疆域快照，源自开源学术地图，涵盖 4,000 年领土变迁。
- **交互式世界地图** 领土叠加层，支持悬停查看与点击进入文明详情。
- **深度文明档案** 每个区域 10 大维度标签页，追踪 100+ 项字段：统治者、政府机构、GDP、贸易品、军事编制、人口结构、文化成就等。
- **战争系统** 结构化冲突追踪，涵盖交战方、开战理由、战略优势、战后影响分析。
- **自定义事件** 注入"假如……"场景，观察 AI 如何应对。
- **时间控制** 播放、暂停、单步、按纪元推进、回滚到任意年份。
- **双语界面** 完整的中英文 UI 与本地化文明数据。

## 运行机制

每次推进时间，模拟器会执行多阶段流水线：

1. **事件生成** —— AI 为下一时段生成历史事件（战争、发明、条约、灾害、迁徙等）。也可注入自定义事件探索架空历史。

2. **区域聚类** —— 编排器基于事件共现、战争交战关系和领土邻近性构建关系图，通过 BFS 提取连通分量。相关文明被分为一组，孤立文明单独批处理。

3. **史学家代理** —— 每组区域被分配给专门的 LLM 提示词：**直接影响**（事件波及区域的完整冲击）、**间接影响**（邻近区域的涟漪效应）或**独立演化**（孤立区域的内生发展）。最多 10 组并行执行，失败自动重试。

4. **状态转移** —— LLM 返回紧凑的 JSON diff（点号路径字段增量），而非完整快照。引擎应用相对增量、绝对赋值和自动计算字段，生成下一世界状态。

5. **战争提取** —— 专用分析器从战争事件中提取结构化冲突数据 —— 交战方、起因、优势、影响 —— 并跨纪元追踪战争进程。

6. **演化日志** —— 每纪元记录逐字段变更日志，驱动历史标签页和演化日志面板。

## 快速开始

### 前置条件

- **Node.js** ≥ 18
- 一个 **[OpenRouter](https://openrouter.ai/)** API 密钥

### 安装与运行

```bash
git clone https://github.com/your-username/human-history-simulator.git
cd human-history-simulator
npm install
```

创建 `.env.local`：

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here
LLM_MODEL=openai/gpt-5.4
LLM_MAX_GROUP_SIZE=10
```

| 变量                 | 说明                                        |
| -------------------- | ------------------------------------------- |
| `OPENROUTER_API_KEY` | OpenRouter API 密钥                         |
| `LLM_MODEL`          | 模拟使用的模型（OpenRouter 上任意可用模型） |
| `LLM_MAX_GROUP_SIZE` | 每次 LLM 调用的最大区域数                   |

```bash
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000) —— 选择纪元，开始模拟。

## 路线图

- [ ] **更智能的推演引擎**：优化历史演进效率，增加可调参数，如「意外性」（偶发事件概率）和「必然性」（结构性力量权重），让用户自由调节混沌与秩序的天平。
- [ ] **更丰富的文明档案**：为每个文明补充更细粒度的数据：社会结构、宗教影响、艺术流派、基础设施等。
- [ ] **更广阔的历史版图**：在地图上呈现更多被忽视但对历史进程有深远影响的国家、部落和地区。教科书遗忘的，历史不会遗忘。
- [ ] **战争影响可视化**：不止于事件日志，实时可视化战争对疆域、人口、经济和权力格局的连锁反应。
- [ ] **更灵活的自定义事件**：让「假如……」注入更具表现力和趣味性：支持事件链、前置条件，构建更精巧的架空历史剧本。
- [ ] **未来纪元推演**：将时间线延伸至当下之后，让推演引擎推测人类的下一章：2050、2100，乃至更远的未来。
- [ ] **视角模式**：化身为一国领袖，透过任意文明的战略视角审视世界：它的威胁、机遇、同盟与盲区。
- [ ] **基于 Skill 的 Agent 接入**：将模拟器封装为一组 Skill，让 OpenClaw 等平台的自主 Agent 代替人类参与交互：选择纪元、注入事件、做出战略决策、驱动模拟持续推进，无需人工操作。
- [ ] **历史经济与金融数据**：为每个时期引入更精确的经济金融数据——贸易总量、货币体系、税收结构、债务水平、财富分配——让文明的经济维度真正扎根于历史研究。
- [ ] **核心资产走势追踪**：建模关键资产在历史长河中的价格轨迹：大宗商品（黄金、白银、粮食、石油）、土地价值、早期股权工具等，以交互式趋势图的形式随模拟同步演进。
- [ ] **精神文明指数**：评估与量化每个时期人类精神文明的状态——哲学思潮、宗教影响力、艺术产出、科学思想、集体精神面貌——作为模拟系统的一等维度纳入追踪。
- [ ] **偶然性 vs. 必然性监控**：加入更细粒度的追踪机制，对比历史偶然性（蝴蝶效应事件、小概率结果）与结构性必然性（地理、经济、人口等深层力量），支持平行推演间的逐项分歧分析。
- [ ] **国家状态实时编辑**：允许在任意时间点直接修改任何文明的状态——调整 GDP、更换统治者、重写同盟关系、调整军事力量——然后观察推演引擎如何向前传播其后果。
